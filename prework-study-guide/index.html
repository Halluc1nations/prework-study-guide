HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • elements, such as the < meta > and < title > elements, are nested inside the < head > element.
  • An HTML developer is responsible for not only providing the structure for the webpage but also making sure that it provides a positive user experience.
  • In semantic HTML elements, the name of the element describes its content and the role it plays on a webpage
  • When possible, it is important to use semantic HTML elements, because they make your HTML easier to read by other developers and the browser.
  • It is still common to see non-semantic elements such as < div > to create a division in the page, or < span > to create an inline division.
  • The < h1 > is the opening tag and < /h1 > is the closing tag.
  • The content between those tags is what is rendered on the page.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • A link at the bottom of the meta data of the head of your HTML is needed to have your CSS changes shown
  • If applying the same attributes to multiple sections such as < h1 > and < h2 > you only need to write that code once.
  • Format goes section title CSS is being applied to { property name: value }
  • class attribute name has to be identified before in every section

Git

  • git status: checks what branch we are currently on
  • git checkout "branch-name": switches to that branch
  • git checkout -b "branch-name": creates a new branch and switches to it
  • git pull origin main:
  • git add -A: stage changes, commit all files in the working branch
  • git commit -m "insert description of adjustment made": Git repository is now up to date with our working directory!
  • Always pull main again before pushing to ensure we have the most up to date main
  • git push origin feature/branch-name: push to github

JavaScript

  • Coding files are processed top down: so for JavaScript to have affect it's link is put at the bottom of the code
  • Note that whereas the style.css file is linked within the < head > element, the script.js file is linked within the < body > element.

  • A semicolon at the end of a line indicates where a statement ends. It is only required when you need to separate statements on a single line.
  • Some people believe it's good practice to have semicolons at the end of each statement
  • JavaScript is case sensitive. This means myVariable is not the same as myvariable
  • There are other rules for when you should and shouldn't use semicolons.
  • You retrieve the value by calling the variable name: myVariable;
  • After assigning a value to a variable, you can change it later in the code: let myVariable = "Bob"; myVariable = "Steve";

  • All other values, including a Boolean with the value of true, are considered truthy.
  • The strict equality operator (===) checks to see if two values are equal, and returns a Boolean result true if the values are equal and false if the values are not equal.
